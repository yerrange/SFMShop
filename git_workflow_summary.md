# Git Workflow для проекта SFMShop

## Использованные команды Git
- git init - инициализация репозитория
- git add - добавление файлов в staging area
- git commit -m 'message' - сохранение снимка состояния проекта
- git merge - слияние двух веток 
- git rebase main - обновление локальной ветки до актуального состояния main
- git push - отправка изменений в удалённый репозиторий
- git pull - обновление локального репозитория данными из удалённого репозитория
- git status - просмотр текущего состояния репозитория
- git log - просмотр истории коммитов
- git reset HEAD^ - откат состояния ветки до предыдущего коммита

## Созданные ветки
- main - основная ветка проекта
- feature/add-discount-system - ветка для системы скидок
- Test_PR - ветка для тестирования pull request
- feature/add-inventory-management_2 - ветка для управления складскими запасами
- feature/add-shipping - ветка для системы доставки
- feature/test - ветка для тестирования различных команд

## Разрешенные конфликты
- Конфликт в src/models/product.py при слиянии feature/add-discount-system

## Стратегия работы с ветками
Для разработки используются отдельные короткие ветки, отделённые от актуального main. По завершении работы в ветке создаётся pull request на GitHub для объединения изменений с веткой main.
